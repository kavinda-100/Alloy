# Alloy

Alloy is a Rust CLI that converts smart contract ABI JSON files into TypeScript type definitions.

It supports both:

- Foundry projects
- Hardhat projects

The tool scans your project for ABI files, lets you pick one interactively, then generates a TypeScript file in the types folder.

## Features

- Auto-detects Foundry or Hardhat project structure
- Prompts when both frameworks are present
- Interactive ABI file selection
- Maps common Solidity types to TypeScript-friendly types
- Writes output to the types directory
- Includes a clean terminal UI with spinner and colored messages

## Solidity to TypeScript Mapping

Current mapping behavior:

- uint*, int* -> bigint
- bool -> boolean
- string, bytes\* -> string
- address -> `0x${string}`
- T[] -> mapped recursively as T[]
- unknown/custom -> any

## Requirements

- Rust toolchain (stable)
- ABI JSON output generated by either Foundry or Hardhat

## Installation

Clone and build:

```bash
git clone https://github.com/kavinda-100/Alloy.git
cd Alloy
cargo build --release
```

Run directly during development:

```bash
cargo run
```

Run release binary:

```bash
./target/release/Alloy
```

## How Framework Detection Works

The CLI checks your current directory:

- Foundry mode: requires foundry.toml and out
- Hardhat mode: requires hardhat.config.ts or hardhat.config.js (and artifacts for js path)
- If both Foundry and Hardhat indicators are found, the CLI asks which framework to use

ABI search roots:

- Foundry: out
- Hardhat: artifacts/contracts

## Usage

From your smart contract project root:

```bash
cargo run
```

Then follow prompts:

1. Select framework if both are detected
2. Select ABI JSON file
3. Confirm or edit output file name

The generated file is saved to:

- types/<OutputName>.ts

## Example

Input ABI file:

- artifacts/contracts/ERC20.sol/ERC20.json

Possible output file:

- types/ERC20AbiTypes.ts

Example generated shape:

```ts
export type ERC20 = {
	transfer(to: `0x${string}`, amount: bigint): Promise<boolean>;
	balanceOf(account: `0x${string}`): Promise<bigint>;
};
```

## Project Structure

```md
├── artifacts/ // Hardhat ABI output
│ └── contracts/
├── out/ // Foundry ABI output
├── src/
│ ├── main.rs
│ └── utils/
│ ├── dtos.rs
│ ├── mod.rs
│ ├── sol_to_ts_mapper.rs
│ └── util.rs
└── types/ // Generated TypeScript files
|-- README.md
|-- .gitignore
|-- Cargo.toml
|-- hardhat.config.ts // Hardhat config
|-- foundry.toml // Foundry config
```

- Important: When Run the project locally, ensure you have either foundry.toml with out directory or hardhat.config.ts/js with artifacts directory in the same root as the CLI for it to detect the framework and find ABI files.

## Notes

- The CLI currently generates function signatures from ABI entries where type is function.
- Constructor, event, and advanced tuple object typing can be added in future updates.
